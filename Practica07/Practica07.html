Objetivo

Aprender a construir y evaluar un modelo de clasificacin de correos

electrnicos como spam o no spam.

Reursos necesarios

  • Acceso al dataset de Kaggle.

  • Herramientas de anlisis. Python con bibliotecas (pandas, numpy, scikit-learn, matplotlib).

Comprensin del Problema

El problema consiste en la mezcla de algunos datasets que contienen la informacin de correos electrnicos etiquetados como spam o no spam, el problema consta de hacer la clasificacin de cuales correos son spam y cuales no, esto ayuda al sistema a clasificar los correos y decidir que correos mostrarle al usuario, los cuales deben de ser los ms relevantes.

Bag of Words

Representa un documento como un conjunto no ordenado de palabras, ignorando la estructura gramatical y el orden de las palabras. Cada palabra nica en el corpus se convierte en una caracterstica, y el valor en cada posicin del vector representa la frecuencia o la presencia de esa palabra en el documento.

Modelo de Clasificacin

Un modelo de clasificacin es un algoritmo o sistema que se entrena para asignar una etiqueta (o clase) a una entrada basndose en ciertas caractersticas o atributos. La tarea principal de un modelo de clasificacin es aprender patrones a partir de datos de entrenamiento para luego hacer predicciones sobre nuevos datos.

  • Datos de Entrada:

Un conjunto de datos de entrada que consiste en ejemplos de entrenamiento, donde cada ejemplo tiene caractersticas y una etiqueta asociada.

  • Entrenamiento del Modelo:

El modelo de clasificacin se entrena utilizando el conjunto de datos de entrenamiento. Durante el entrenamiento, el modelo ajusta sus parmetros internos para aprender los patrones que relacionan las caractersticas de entrada con las etiquetas de salida.

  • Predicciones:

Despus del entrenamiento, el modelo puede realizar predicciones sobre nuevos datos. Dada una entrada sin etiqueta, el modelo asigna una etiqueta basndose en los patrones aprendidos durante el entrenamiento.

  • Evaluacin del Rendimiento:

El rendimiento del modelo se evala utilizando un conjunto de datos de prueba o de validacin que no se utiliz durante el entrenamiento. Mtricas comunes de evaluacin incluyen precisin, recall, F1-score, matriz de confusin, entre otras, dependiendo de la naturaleza del problema.

Matriz de Confusin y Mtricas de Evaluacin

  • Matriz de confusin

Una matriz de confusin es una tabla que se utiliza en clasificacin para evaluar el rendimiento de un modelo. Esta matriz compara las predicciones del modelo con los valores reales y resume la cantidad de verdaderos positivos (TP), falsos positivos (FP), verdaderos negativos (TN) y falsos negativos (FN).

Verdaderos Positivos (TP): 
Casos en los que el modelo predijo positivo y la clase real tambin es positiva.

Falsos Positivos (FP): 
Casos en los que el modelo predijo positivo, pero la clase real es negativa.

Verdaderos Negativos (TN): 
Casos en los que el modelo predijo negativo y la clase real tambin es negativa.

Falsos Negativos (FN): 
Casos en los que el modelo predijo negativo, pero la clase real es positiva.
  • Exactitud

Proporcin de predicciones correctas con respecto al total de predicciones. Es una medida general del rendimiento del modelo.

$$ \frac{TP+TN}{TP+FP+TN+FN} $$
  • Precisin

Proporcin de instancias positivas predichas correctamente entre todas las instancias positivas predichas.

$$ \frac{TP}{TP+FP} $$
  • Tasa de error

Proporcin de predicciones incorrectas con respecto al total de predicciones.

$$ \frac{FP+FN}{TP+FP+TN+FN} $$
  • Sensibilidad (recall)

Proporcin de instancias positivas predichas correctamente entre todas las instancias positivas reales.

$$ \frac{TP}{TP+FN} $$
  • Especificidad

Proporcin de instancias negativas predichas correctamente entre todas las instancias negativas reales.

$$ \frac{TN}{TN+FP} $$
  • Comprensibilidad

Media armnica de precisin y sensibilidad. Es til cuando hay un desequilibrio entre las clases.

$$ \frac{2 * Presicion * Sensibilidad}{Presicion + Sensibilidad} $$

Evaluacin del Desempeo

  • Mtodo de Retencin (Holdout):

Divide el conjunto de datos en conjuntos de entrenamiento y prueba. El modelo se entrena en el conjunto de entrenamiento y se evala en el conjunto de prueba.

  • Validacin Cruzada (Cross-Validation):

Divide el conjunto de datos en k pliegues (folds). El modelo se entrena k veces, cada vez utilizando k-1 pliegues como conjunto de entrenamiento y el pliegue restante como conjunto de prueba.

  • Submuestreo Aleatorio (Random Undersampling):

Reduce el tamao de la clase mayoritaria al eliminar aleatoriamente ejemplos. Se utiliza en problemas de desequilibrio de clases.

  • Bootstrap:

Muestreo con reemplazo. Se generan mltiples conjuntos de datos bootstrap a partir del conjunto de datos original, y el modelo se entrena y evala en cada conjunto.

  • Describe el proceso de preprocesamiento de datos aplicado al conjunto de correos electrnicos.

El proceso de "Bag of Words" (BoW) es una tcnica fundamental en procesamiento de lenguaje natural (PLN) y se utiliza comnmente para representar documentos de texto de manera numrica. Aqu hay una descripcin del proceso:

Tokenizacin:

El primer paso es dividir el texto en unidades ms pequeas llamadas "tokens". Los tokens pueden ser palabras individuales o incluso caracteres, dependiendo de la granularidad deseada.

Creacin del Vocabulario:

Se crea un vocabulario nico que consiste en todas las palabras nicas encontradas en el conjunto de documentos. Cada palabra en el vocabulario se convierte en una "feature" o caracterstica.

Creacin del Vector (BoW):

Para cada documento, se crea un vector que tiene la longitud del vocabulario. Cada posicin en el vector corresponde a una palabra en el vocabulario, y el valor en esa posicin indica cuntas veces aparece esa palabra en el documento. Puede ser simplemente un conteo de ocurrencias.

Ignorar la Estructura y el Orden:

La principal caracterstica del BoW es que ignora completamente la estructura gramatical y el orden de las palabras en el documento. Solo se tiene en cuenta la frecuencia de las palabras.

Matriz BoW:

Cuando se aplica el proceso a un conjunto de documentos, se obtiene una matriz BoW. Cada fila de la matriz representa un documento y cada columna representa una palabra en el vocabulario.

Frecuencias o Binario:

En lugar de contar la frecuencia de las palabras, tambin se puede usar un enfoque binario, donde la presencia o ausencia de una palabra en el documento se marca con 1 o 0, respectivamente.

Exactitud

Proporcin de predicciones correctas con respecto al total de predicciones. Es una medida general del rendimiento del modelo.

$$ \frac{TP+TN}{TP+FP+TN+FN} $$

Precisin

Proporcin de instancias positivas predichas correctamente entre todas las instancias positivas predichas.

$$ \frac{TP}{TP+FP} $$

Tasa de error

Proporcin de predicciones incorrectas con respecto al total de predicciones.

$$ \frac{FP+FN}{TP+FP+TN+FN} $$

Sensibilidad (recall)

Proporcin de instancias positivas predichas correctamente entre todas las instancias positivas reales.

$$ \frac{TP}{TP+FN} $$

Especificidad

Proporcin de instancias negativas predichas correctamente entre todas las instancias negativas reales.

$$ \frac{TN}{TN+FP} $$

Comprensibilidad

Media armnica de precisin y sensibilidad. Es til cuando hay un desequilibrio entre las clases.

$$ \frac{2 * Presicion * Sensibilidad}{Presicion + Sensibilidad} $$

Impacto del Desequilibrio de Clases:

Sesgo del Modelo:

El modelo puede desarrollar un sesgo hacia la clase dominante, ya que tiene ms ejemplos para aprender patrones. Esto puede llevar a que la clase minoritaria se ignore o tenga un rendimiento deficiente.

Baja Sensibilidad a la Clase Minoritaria:

El modelo puede tener una baja sensibilidad (recall) para la clase minoritaria, lo que significa que no identifica correctamente muchos de los casos positivos.

Precisin Engaosa:

La precisin puede ser engaosa en conjuntos de datos desequilibrados. Un modelo que predice siempre la clase dominante puede tener una alta precisin simplemente porque acierta la mayora de las instancias de esa clase.

Estrategias para Abordar el Desequilibrio de Clases:

Remuestreo:

Submuestreo (Undersampling): Reduce el nmero de instancias de la clase mayoritaria para equilibrar proporciones.

Ponderacin de Clases:

Asigna pesos diferentes a las clases durante el entrenamiento para penalizar ms los errores en la clase minoritaria.

Mtodos de Evaluacin Especficos:

Utiliza mtricas como precisin, recall y F1-score en lugar de la exactitud, ya que son ms informativas en conjuntos de datos desequilibrados.

Enfoques de Modelos Especficos:

Algunos algoritmos de aprendizaje automtico tienen parmetros especficos para manejar el desequilibrio, como el parmetro class_weight en Scikit-Learn para ciertos modelos.

Ensamble de Modelos:

Utiliza tcnicas de ensamblado como Bagging y Boosting con modelos base que manejen el desequilibrio de clases de manera efectiva.

Generacin de Caractersticas Informativas:

Crea caractersticas adicionales que ayuden al modelo a distinguir mejor entre las clases, incluso si la cantidad de datos es limitada.

Validacin Cruzada Estratificada:
